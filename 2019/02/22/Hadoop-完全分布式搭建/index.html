[Hadoop]-完全分布式搭建

发表于 2019-02-22 | 分类于 Hadoop

用mapreduce实现了基于蚁群算法的TSP问题（旅行商问题），在单机调试环境和伪分布式运行过后，希望在完全分布式上运行一下。

操作系统：macOS Mojave OS 10.14.1

虚拟机：VMware-Fusion-11.0.2

master：macOS Mojave OS 10.14.1

slave1： ubuntu-18.04.1-desktop-amd64.iso 64位

slave2： ubuntu-18.04.1-desktop-amd64.iso 64位

hostName IP OS Accounts role
master1 192.168.1.106 macOS Mojave OS 10.14.1 namenode
slave1 192.168.1.110
slave2 192.168.1.108

查看ip,修改hostname

查看ip

ifconfig查看三台机器的ip地址，若没有这个指令，安装：sudo apt install net-tools

设置静态ip

时候虚拟机重启后发现虚拟机的IP发生了变化，这就导致了hosts的设置失效，必须重新设置hosts。

参照本文设置静态ip

https://blog.csdn.net/Yaphat/article/details/53007127

修改hostname

将三台机器的hostname改为master1，slave1，slave2

查看hostname：hostname

linux修改hostname：sudo vim /etc/hostname,修改hostname文件，注意不同的发行版修改文件位置可能不同。

mac修改hostname：sudo scutil --set HostName master1

添加host解析，sudo vim /etc/hosts,给每一个节点都添加上域名解析，以slave2为例

至此，host设置完成，并且相互之前ping hostname可以ping通

设置ssh免密登陆

每个节点安装好ssh服务后，各自生成密钥对 ssh-keygen -t rsa

在master上，导入authorized_keys: cat ~/.ssh/id_dsa.pub >> ~/.ssh/authorized_keys

将改通证复制到两个slave上：

所有机器上，修改这个文件的权限。均执行命令：chmod 600 .ssh/authorized_keys

测试免密登陆是否成功：ssh slave1/slave2
